# Security Policy

Nemiga Tower is maintained as a public demo repository. Even so, security reports should be handled privately whenever possible.

## Supported Versions

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Historical commits and snapshots | No |

## Reporting A Vulnerability

- Do not open a public GitHub issue for a suspected vulnerability.
- If private vulnerability reporting is enabled for this repository, use that path first.
- If it is not available, contact the maintainer privately through the public contact method listed on the GitHub profile and include `Nemiga Tower security report` in the subject or message.

Please include:

- a short description of the issue
- affected route or feature
- reproduction steps
- potential impact
- any suggested mitigation if you already tested one

## Response Expectations

- Reports are reviewed on a best-effort basis.
- There is no formal SLA for this demo repository.
- Public disclosure should wait until the maintainer confirms triage or mitigation guidance.
